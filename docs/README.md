# Pupperfish docs

This directory contains the practical documentation for the public `pupperfish` monorepo.

## Read this first
- [Architecture](architecture.md): how the monorepo is split and where responsibilities stop.
- [API reference](api-reference.md): the contracts, runtime methods, and React surfaces you integrate against.
- [Deployment and release guide](deployment.md): how to build, smoke-test, and publish the packages.
- [Troubleshooting](troubleshooting.md): the common failures when wiring a host app to Pupperfish.
- [Getting started tutorial](tutorials/getting-started.md): the smallest viable integration path.

## Package entry points
- [Root README](../README.md)
- [Framework package README](../packages/pupperfish-framework/README.md)
- [React package README](../packages/pupperfish-react/README.md)

## Reader map
- If you are evaluating the repo: start with the root `README.md`, then `architecture.md`.
- If you are integrating the runtime: go to `api-reference.md` and `tutorials/getting-started.md`.
- If you are preparing a release: go to `deployment.md`.
- If you hit build, client, or CSS issues: go to `troubleshooting.md`.
